📁 Step 1 — GitHub Repository

Upload the project to GitHub so Netlify can pull it and auto-deploy every time you make a change.

📤 Create a new GitHub repo and upload files
1
Go to github.com → sign in → click New repository (green button, top right).
2
Name it expensio → set to Public (required for free Netlify) → Create repository.
3
On the next screen click "uploading an existing file" link.
4
Extract the ZIP you downloaded → drag all files and folders into the upload area. Make sure netlify/functions/ folder is included.
5
Click Commit changes → your code is now on GitHub.
📂 Your repo should contain these files
expensio/ ├── index.html # the app ├── manifest.json ├── sw.js ├── netlify.toml # deploy config ├── README.md ├── icons/ │ └── icon-*.png └── netlify/functions/ ├── config.js # serves Firebase keys └── pay.js # Razorpay + verification

🔥 Step 2 — Firebase Project

Firebase handles Google login and stores each user's license status (trial / pro / expired). It's free for thousands of users.

🏗 Create the project
1
Go to console.firebase.google.com → click Add project.
2
Name it expensio-app → Continue → disable Google Analytics → Create project.
🔑 Enable Google Sign-In
3
Left sidebar → Build → Authentication → Get started.
4
Sign-in method tab → click Google → toggle Enable → set support email → Save.
🗄 Create Firestore Database
5
Left sidebar → Build → Firestore Database → Create database.
6
Choose Production mode → Next → region: asia-south1 (Mumbai) → Enable.
7
Click Rules tab → replace everything with this and click Publish:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /licenses/{userId} {
      allow read, write: if request.auth != null
        && request.auth.uid == userId;
    }
  }
}
📋 Get your API keys (you'll paste these into Netlify)
8
Click the ⚙ gear icon (next to "Project Overview") → Project settings.
9
Scroll to Your apps → click the </> web icon → nickname: expensio-web → Register app.
10
You'll see a block with 6 values — keep this tab open, you'll paste them in Step 5.

💳 Step 3 — Razorpay Account

Razorpay processes payments in India — UPI, cards, and net banking. Free to set up; they take 2% per transaction.

🏦 Create account
1
Go to razorpay.com → Sign up with your email.
2
Complete KYC: enter business name, PAN, and bank account. This takes 1-2 business days to verify.
🔑 Get API keys
3
Dashboard → Settings → API Keys.
4
Click Generate Test Key first (for testing). You'll get two values:

• Key ID — starts with rzp_test_... (safe to share, goes in Netlify env vars)
• Key Secret — starts with a long string (keep this private, never put in code)
5
Enable payment methods: Settings → Payment Methods → enable UPI, Credit Card, Debit Card, Net Banking → Save.
🔒
Important: You have two Razorpay keys — the Key ID and the Key Secret.
The Key ID is safe to store in Netlify env vars and gets sent to the browser.
The Key Secret must stay on the server only — it's what signs and verifies payments. Never paste it in your HTML.

🌐 Step 4 — Netlify Site

Connect Netlify to your GitHub repo. Every time you push code, Netlify rebuilds and deploys automatically.

🔗 Connect GitHub → Netlify
1
Go to app.netlify.com → sign in → click Add new site → Import an existing project.
2
Click GitHub → authorise Netlify → select your expensio repo.
3
Build settings are auto-detected from netlify.toml. Just click Deploy site.
4
Your site URL will be something like expensio-abc123.netlify.app. Copy this — you'll need it in Firebase and env vars.
🔒 Add your domain to Firebase
5
Back in Firebase Console → Authentication → Settings → Authorized domains → Add domain.
6
Add expensio-abc123.netlify.app (your actual Netlify URL) and also add localhost.

🤖 Step 6 — Automatic Deployment

Once set up, every time you push a change to GitHub, Netlify automatically rebuilds and deploys. No manual uploads ever again.

🔑 Get Netlify tokens
1
Netlify Personal Access Token:
Netlify → click your avatar (top right) → User settings → Applications → Personal access tokens → New access token → copy the token.
2
Netlify Site ID:
Netlify → your site → Site configuration → General → Site ID → copy the ID.
🔐 Add GitHub Secrets
3
GitHub repo → Settings → Secrets and variables → Actions → New repository secret.
4
Add secret: name = NETLIFY_AUTH_TOKEN, value = your personal access token.
5
Add secret: name = NETLIFY_SITE_ID, value = your site ID.
✅ How it works from now on
Edit any file locally (e.g. change Pro price in netlify.toml)
Run: git add . && git commit -m "update price" && git push
GitHub Actions detects the push → runs deploy.yml → deploys to Netlify
Live in ~60 seconds. No manual steps.

3
